Q: What is bbc_to_faq?

A:

bbc_to_faq is a toolset designed to replicate at least the functionality of BBcode on the forums. It is intended to allow you to write preliminary FAQ items in a more familiar language and with less repetitive overhead. It is also intended to allow the outlines of FAQ items to be quickly written out with less concern on the specifics of the formatting.

To the best of its ability, the toolset will attempt to generate code which is at least technically correct for an FAQ item, and which should require significantly less work to clean up than writing it in full in the first place would be.

Q: Is there any difference between forum BBcode and the BBcode used by bbc_to_faq?

A:

Yes.
bbc_to_faq is intended to generate FAQ items, and is designed to automatically generate output which is compatible with that purpose. Though it is designed to look and function mostly like BBcode (for the sake of familiarity), it is designed to output complete FAQ items (as opposed to posts on forum). To this end, it has several tags which will never be implemented on the forums, because they allow too much flexibility and/or present legitimate security risks. These risks are identical to the risks associated with allowing posting directly in HTML, which is fine for the TB-FAQ project, but would be a very bad idea on the forums.

Thus, some codes may seem to behave similarly to the forum's BBcodes, but several of them also have additional features. For example, the bold and italics tags, [b][/b] and [i][/i], in this toolset will accept an option parameter that will be passed to their style attribute. For example:
[b="color:red"]Example[/b] becomes: Example

One other key difference is that "Q:" and "A:" are special dividers which represent question and answer segments. These will be automatically formatted before any BBcode is processed, and BBcode cannot be transferred across these segments.

Q: What tags (BBcodes) does bbc_to_faq support?

A:

Not many at the moment, though adding more is usually quite simple.

Supported tags ►

  • [b]: Bold text. Accepts a CSS styling parameter as its option.
  • [i]: Italic text. Accepts a CSS styling parameter as its option.
  • [list]: Generates a list, separated by [*].
    • A
    • B
    • C
  • [url]: Links.
  • [img]:
    Note that it accepts a hover-text as its option.
  • [dropdown]:
    Dropdowns ►

    They drop down.

  • [important]:

    Highlight important information.

  • [snippet]: Code snippets.
  • [table]:
    TurnToriUke
    100
    2999,999,9990
    3999,999,999-1
  • [noparse]: Text within a [noparse] tag will not be parsed as BBcode.
  • [p]: Can be used to explicitly specify a paragraph block.
  • [hover]: Hold mouse over this... Requires the option parameter.

Q: I need more tags!

A:

Contact suomy on IRC, make an issue, or add them yourself. See bbcode_parser.py if you're interested on how to do that. In the case of standard tags, they will be added as they seem necessary, so if you want to prioritize a tag, use it in an FAQ item. Even if it doesn't work now, it will work by the time we go to merge it.

Q: How do I use the bbc_to_faq tool to generate the HTML?

A:

You will need Python (it was developed on Python 3.6.3, but lower versions of Python 3 may work). Then, open python in the same directory, from bbc_to_faq import make_html_from_file, and then make_html_from_file(input_filename,output_filename).
