Operator Commands

Q: What are operator commands?

A: This category includes commands which can only be run as a server operator (@). In multiplayer, set commands may also only be executed by operators.

Some operator commands may also be run in singleplayer.

Use chat commands like so: /<command> <argument(s)>
Some commands may not have arguments:
Example no-argument command: /specall
Example command with arguments: /fspec Fear

Other command types:

  • Standard commands
  • Option commands
  • Set commands

Q: Where can I see all the commands?

A: Here's a full list of commands only usable by room operators, including descriptions of what they do:

Commands in green may also be used in singleplayer.
Some of these commands may not work in certain versions of Toribash

Room control commands
  • COMMAND ARGS SHORTHAND DESC
    passwd [password|none] Adds or removes a password for the room: users must use /password [password] to enter and chat.
    Running this command with no argument will clear the password entirely.
    setowner [password] Sets the ownerpass for the room.
    Use in conjunction with /owner [password].
    owner [password] If the password is correct, you will be given operator status by the server.
    reset rt Resets the game
    loadmod [filename].tbm lm Loads the specified mod file (.tbm file) and automatically resets the game.
    Equivalent to calling /set mod [modname].tbm then /reset.
    desc [text] Changes the description of the room as seen on the server list.
    motd [text] Changes the 'message of the day' of the room: This message will be displayed to people joining your room.
    maxclients [number] Sets the maximum number of users (clients) that can join your room.
    minbelt [qi] Sets the minimum number of games played (qi) a user must have to join your room.
    maxbelt [qi] Sets the maximum number of games played (qi) a user can have to join your room.
    queuejump [position] Users joining your room will be automatically nudged to the specified position in the queue:
    1. End of queue (default setting).
    2. Uke's position.
    3. First position in queue.
    4. Place new users into specified place.
      If the number is higher than the queue size, they will be added to the end of the queue.
    duel
    • [start][TC]
    • [stop|status|earnings]
    		 See dueling.
    lockdown [on|off]

    		 Imposes lockdown mode on the room: Users in spectator mode won't be able to join the bout queue by themselved.
    If a user reconnects, they will automatically join the bout queue as normal, however.
    knockout [on|off] Imposes knockout mode on the room: Users will be specced upon losing, and cannot re-enter until the tourney is over.
    Users joining the room will be automatically placed into spectator mode.
    The winner may automatically win a prize generated by the server in the case of autotourneys, or from another user's account in the case of /knockoutprize [TC].
    knockoutprize [TC] Winners of a knockout tourney hosted in the room will receive the specified amount of TC, from your own TC.
    hotseat [on|off]

    		 Imposes hotseat mode on the room: The player in Tori's position will always be in that position regardless of whether they win or lose their matches.
    As such, the player in Uke's position will be sent to the end of the queue regardless of whether they win or lose their match.
    status

    		Display the IP addresses of all users in the room.
    centerprint [message] cp Displays the specified message in the center of the screen for all users in the room.

User control commands
  • COMMAND ARGS SHORTHAND DESC
    operator [username] op Makes the specified user operator in the room.
    deoperator [username] deop Removes operator status from the specified user in the room.
    fenter [username] Moves the specified user from spectator mode into the bout queue.
    fspec [username] Moves the specified user from the bout queue into the spectator mode.
    fknock [username] Moves the specified user to the bottom of the bout queue (must already be in the bout queue).
    specall Moves all players in the room to spectator mode.
    scramble Randomizes the positions of all players in the bout queue.
    nudge [username][position] Nudges the specified user to the specified position in the queue: If the position is higher than the size of the queue, they will be nudged to the end of the queue.
    Cannot be used to nudge people into the position of Tori or Uke.
    Players must be in the bout queue to be nudged.
    mute [username] Causes the specified user to be unable to chat.
    If the user reconnects, this effect will be removed.
    muteall Causes all users in the room to be muted.
    If a user reconnects, this effect will be removed from that user.
    unmuteall Causes all users in the room to be unmuted.
    kick [username] Causes the specified user to be disconnected from the room.
    You cannot kick ingame admins.
    ban
    • [list|clear]
    • [add|del][username|ip]
    		 Banned users cannot rejoin a room after being kicked or otherwise disconnecting (does not affect ingame admins).
    Usage:
    • list: lists all users on the room's banlist.
    • clear: clears the entire banlist.
    • add|del: adds/removes the specified user to/from the banlist.
    Ban a user's IP to prevent them from rejoining on an alternate account (find IPs with /status).

Betting control commands
  • COMMAND ARGS SHORTHAND DESC
    house [username|none] Makes the specified user server admin: This doesn't actually grant any special privileges, rather, the user will earn tc from certain other commands such as /maxrake.
    minbet [TC] Forces all bets to be at least the specified amount.
    betframes [frames] All bets must occur before the specified amount of frames have elapsed.
    maxrake [TC] Specifies the maximum amount of rake TC the server admin can earn from bets.
    Note that rake TC is calculated and you will not always earn this amount -- it is just a maximum.
    cancelbets Cancels all active bets, returning all TC.
    tourneystats Displays info on house earnings and bet statistics.
